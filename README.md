# Zero-Trust WASM Sandbox (Clean Sandwich Architecture)

This repository contains a localized Proof of Concept (POC) demonstrating **"Clean Sandwich Architecture"**—a highly secure method to isolate language-model generated code dynamically without risking the host machine.

## Architecture Highlights
The core philosophy is based on providing "Zero-Trust" guarantees for AI-generated code.
1. **The Python Host (`agent.py` & `server.py`)**: An `atomic-agents` powered LLM orchestration engine running over FastAPI. It leverages `instructor` and `litellm` in strict JSON mode to ensure LLM outputs strictly match predefined Pydantic schemas without falling into Tool-Calling hallucination loops. It natively serves a real-time web dashboard using WebSockets/SSE to emit deep architectural telemetry traces.
2. **The WASM Bridge (`sandbox_manager.js`)**: A strict, zero-dependency Node.js execution boundary. It securely isolates Pyodide WebAssembly (WASM) inside `worker_threads` and natively sanitizes double-escaped strings (`\\n`) generated by local LLMs before compiling them.
3. **Implicit Clean-Up Strategy**: When the WASM worker finishes processing the LLM-generated Python code, the main Node thread **explicitly destroys** the worker instance, preventing any lingering allocations, background loops, or memory leaks before returning the structured JSON execution format back to the Host.

This removes the need for containerized dockers, allowing for light, seamless, and instantaneous AI workflow execution entirely local to the project directory.

## Getting Started

Because the Node.js bridge leverages a native Pyodide CDN downloader, it has structurally zero `node_modules`. All operations require configuring the Python host environment.

### 1. Initialize the Environment
> [!IMPORTANT]
> The python dependencies must be installed safely inside the local `.venv`. 

**For macOS/Linux:**
```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Configure Your LLM (Optional)
Out of the box, `agent.py` connects locally to Ollama.
If you prefer mistral, or wish to use cloud providers (Gemini, Claude, GPT), create a `.env` file in this directory and specify your model (ensure you include strict tags like `:70b` if running heavier local models):

```env
LLM_MODEL=ollama/llama3.1:70b
# Or for Claude:
# LLM_MODEL=claude-3-opus-20240229
# ANTHROPIC_API_KEY=your_key_here
```

### 3. Start the Dashboard Server
To ensure you are picking up the required dependencies perfectly without dealing with terminal path caching, start explicitly via the `.venv`:

```bash
.venv/bin/python -m uvicorn server:app --port 8000
```

### 4. Observe the Detailed Telemetry Sandbox Lifecycle
* Navigate to `http://127.0.0.1:8000` in your browser.
* Click **Deploy LLM Agent**.
* Watch the exact UI telemetry as it streams the internal architecture states:
    - Live bindings of `Pydantic` schemas for Data Contracts.
    - Full outputs of `SystemPromptGenerator` guidelines.
    - Live payload passing into the WASM Bridge Node.js Sandbox.
    - Secure worker memory destruction.
